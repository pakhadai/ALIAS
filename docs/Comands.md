Ось найкорисніші команди, згруповані за призначенням, які ідеально підійдуть для вашого TypeScript/React/Node.js монорепозиторію:

🗺 1. Планування та Архітектура (Перед тим, як писати код)
/plan — Створює детальний план реалізації фічі. ШІ продумає архітектуру, скаже, які файли треба змінити чи створити, і як вони будуть взаємодіяти. Ідеально для великих задач.

/update-codemaps — Просить ШІ проаналізувати поточну структуру проекту і створити (або оновити) файл з архітектурою. Це допомагає самому ШІ краще орієнтуватися у вашому коді в майбутньому.

/multi-plan — Потужніша версія /plan для складних систем, коли треба скоординувати роботу бекенду (server) та фронтенду (client).

💻 2. Написання коду та виправлення (У процесі роботи)
/tdd (Test-Driven Development) — Змушує Cursor спочатку написати тести (наприклад, для логіки у server/src/services), а вже потім писати сам код, поки тести не пройдуть.

/build-fix — Якщо TypeScript кидає помилки компіляції або щось зламалося після злиття гілок, ця команда викличе агента, який знайде причину і запропонує виправлення.

/refactor-clean — Просить ШІ знайти і видалити "мертвий" (невикористовуваний) код, зайві імпорти або спростити заплутану логіку у файлі.

🕵️‍♂️ 3. Огляд коду та Якість (Після написання коду)
/code-review — Викликає суворого віртуального сеньйора. Він перевірить виділений код на відповідність стандартам, безпеку, продуктивність (наприклад, зайві ререндери в React) та типізацію.

/quality-gate — Глобальна перевірка. ШІ перевірить, чи код готовий до коміту (чи проходять лінтери, чи немає забутих console.log або TODO).

🧪 4. Тестування (Playwright / Jest)
/e2e — Спеціальна команда для генерації End-to-End тестів (наприклад, для вашого пакету packages/e2e). ШІ напише тест на Playwright, використовуючи правильні патерни (Page Object Model).

/test-coverage — Аналізує, яких тестів не вистачає для конкретного файлу чи компонента, і дописує їх, щоб підняти відсоток покриття.

/verify — Запускає цикл перевірки (Build -> Test -> Lint), щоб переконатися, що все працює як слід.

📚 5. Документація
/update-docs — Оновлює README.md або JSDoc-коментарі у коді відповідно до останніх змін, які ви внесли.

🧠 6. Навчання асистента (Фішка ECC)
/learn — Якщо ви довго з ШІ вирішували якусь специфічну проблему (наприклад, хитрий баг із Socket.io), напишіть цю команду. ШІ "витягне" урок із вашої бесіди і запам'ятає цей патерн на майбутнє.

/skill-create — Аналізує вашу історію Git і створює нову "навичку" (Skill) на основі того, як ви зазвичай пишете код.

Як це все використовувати?
Просто відкрийте Composer (чат Cursor) і напишіть команду разом із вашим завданням.
Наприклад: /code-review Перевір цей LobbyFlow.tsx, чи все ок з хуками React?
